Complexity theory

Once we know what is computable, the next question is what is computable cheaply. Time and space, big-O and big-Omega, P and NP, polynomial-time reductions, the Cook–Levin tableau, Savitch's recursive simulation, and the diagonal hierarchy theorems — six concepts that carve the feasible from the infeasible.

Computability fixed the boundary between solvable and unsolvable. Complexity refines the solvable side: how do resource costs (time, memory, randomness, advice) scale with input size? The headline class is $\mathsf{P}$ — problems decidable by a deterministic Turing machine in polynomial time — paired with $\mathsf{NP}$, problems with a polynomial-time verifier. Whether $\mathsf{P}=\mathsf{NP}$ is the deepest open problem in the field. Around this central drama sit the rest of the cast: polynomial-time reductions, NP-completeness, PSPACE, the hierarchy theorems, and the few separations we can actually prove.

Plan. Section 1 fixes asymptotic notation and the time class $\mathrm{DTIME}$. Section 2 introduces $\mathsf{P}$ and $\mathsf{NP}$ via the verifier definition. Section 3 establishes Cook–Levin and the notion of NP-completeness. Section 4 walks the Karp web — how 3-SAT propagates hardness to clique, vertex-cover, Hamiltonian cycle, subset-sum. Section 5 does the same for space: $\mathsf{PSPACE}$, Savitch, TQBF. Section 6 turns the diagonal back on time and space themselves to prove the only strict separations we know.

1. Time complexity and big-O

Fix a multitape Turing-machine model. The running time $t_M(x)$ of $M$ on input $x$ is the number of transitions before $M$ halts. The time class is

\[ \mathrm{DTIME}(f(n)) \;=\; \{\,L : \exists\,M\text{ deciding }L\text{ with }t_M(x)\le f(|x|)\text{ for all }x\,\}. \]

Constants and lower-order terms are absorbed by asymptotic notation:

  • Big-O. $f=O(g)$ iff $\exists c>0,\,n_0$ with $f(n)\le c\,g(n)$ for $n\ge n_0$.
  • Big-Omega. $f=\Omega(g)$ iff $g=O(f)$.
  • Big-Theta. $f=\Theta(g)$ iff $f=O(g)$ and $f=\Omega(g)$.
  • Little-o. $f=o(g)$ iff $f(n)/g(n)\to 0$.
Polynomial vs exponential. $\mathsf{P}=\bigcup_{k\ge 1}\mathrm{DTIME}(n^k)$ is the working notion of feasible. $\mathsf{EXP}=\bigcup_{k\ge 1}\mathrm{DTIME}(2^{n^k})$. The gap between $n^{100}$ and $2^n$ is enormous in practice but feels small on a log-log plot — the widget makes this concrete.

2. P and NP

A decision problem is a language $L\subseteq\{0,1\}^*$. We ask: given $x$, is $x\in L$?

$\mathsf{P}=\bigcup_k\mathrm{DTIME}(n^k)$ is the class of languages with a polynomial-time decider. $\mathsf{NP}$ is the class with a polynomial-time verifier:

\[ L\in\mathsf{NP} \iff \exists\,V\in\mathsf{P},\,p\in\mathbb{Z}[x]\text{ with } x\in L \iff \exists w,|w|\le p(|x|),\,V(x,w)=1. \]

The witness $w$ is a short certificate that $x$ is a yes-instance, and $V$ checks it in polynomial time.

Trivially $\mathsf{P}\subseteq\mathsf{NP}$ — a decider is a verifier that ignores the witness. The reverse inclusion is the $\mathsf{P}$ vs $\mathsf{NP}$ question: does every short certificate translate into an efficient algorithm? Almost everyone believes no, but no proof exists.

Equivalent formulation. $\mathsf{NP}=\bigcup_k\mathrm{NTIME}(n^k)$, where $\mathrm{NTIME}(f)$ uses a nondeterministic TM that on each step may branch and accepts iff some branch reaches $q_{\text{acc}}$ within $f(n)$ steps. The branch leading to acceptance is exactly the witness.

3. NP-completeness and Cook–Levin

A language $L$ is NP-hard (under polynomial-time many-one reductions) if every $A\in\mathsf{NP}$ admits a polynomial-time computable $f$ with $x\in A \iff f(x)\in L$. It is NP-complete if additionally $L\in\mathsf{NP}$. NP-complete problems are the hardest in $\mathsf{NP}$: a polynomial-time algorithm for any one would collapse $\mathsf{P}=\mathsf{NP}$.

Cook (1971), Levin (1973). SAT — boolean satisfiability — is NP-complete.

The proof builds, from any nondeterministic polynomial-time machine $M$ and input $x$, a polynomial-size CNF $\varphi_{M,x}$ such that $\varphi_{M,x}$ is satisfiable iff $M$ accepts $x$. The construction lays out a tableau: rows are time steps $0,1,\ldots,T(n)$, columns are tape cells $1,\ldots,T(n)$, each cell holds a symbol or a (state, symbol) pair when the head is there. Variables $z_{i,j,s}$ encode "cell $j$ at time $i$ holds $s$". Clauses force:

  • each cell holds exactly one symbol (uniqueness);
  • row $0$ matches the input $x$ (initialisation);
  • each $2\times 3$ window in adjacent rows respects $\delta$ (transition);
  • some row reaches $q_{\text{acc}}$ (acceptance).

Each constraint is a constant-size clause; total size is $\mathrm{poly}(T(n))=\mathrm{poly}(n)$. A satisfying assignment is exactly an accepting computation history.

4. Karp reductions and complete problems

Once SAT is NP-complete, the cheapest way to show another problem $L$ is NP-complete is to find any known NP-complete $A$ with $A\le_p L$. Karp's 1972 paper kicked off the bestiary by reducing SAT to 21 natural combinatorial problems; today the catalogue runs to thousands.

Two illustrative reductions:

  • 3-SAT $\le_p$ CLIQUE. Given a 3-CNF $\varphi$ with $m$ clauses, build a graph $G$: one vertex per literal in each clause, edges between literals from different clauses unless they are negations of each other. A satisfying assignment picks one literal per clause, all mutually consistent — exactly an $m$-clique.
  • CLIQUE $\le_p$ INDEPENDENT-SET. Take the complement graph $\overline G$. A clique in $G$ is an independent set in $\overline G$ of the same size. (Trivially also $\le_p$ in the other direction.)
Transitivity. $\le_p$ is closed under composition: if $f$ and $g$ are polynomial-time, so is $g\circ f$. So the Karp web propagates hardness wherever an arrow lands.

5. Space complexity and PSPACE

Define $\mathrm{DSPACE}(f)$ and $\mathrm{NSPACE}(f)$ analogously to time, but counting the number of distinct tape cells touched (excluding the read-only input tape — so sub-linear space classes are meaningful). The standard hierarchy is

\[ \mathsf{L} = \mathrm{DSPACE}(\log n) \subseteq \mathsf{NL} = \mathrm{NSPACE}(\log n) \subseteq \mathsf{P} \subseteq \mathsf{NP} \subseteq \mathsf{PSPACE} = \bigcup_k\mathrm{DSPACE}(n^k) \subseteq \mathsf{EXP}. \]

The simulation $\mathsf{P}\subseteq\mathsf{PSPACE}$ is trivial; $\mathsf{NP}\subseteq\mathsf{PSPACE}$ comes from enumerating all witnesses one at a time. The space side has a striking equality:

Savitch (1970). For $f(n)\ge\log n$, $\;\mathrm{NSPACE}(f)\subseteq\mathrm{DSPACE}(f^2)$. Hence $\mathsf{PSPACE}=\mathsf{NPSPACE}$.

Idea. A nondeterministic $f$-space machine has at most $2^{O(f)}$ configurations. To test whether $C_{\text{start}}$ reaches $C_{\text{accept}}$ in $\le 2^k$ steps, recursively ask whether some midpoint $C'$ is reachable in $\le 2^{k-1}$ steps from $C_{\text{start}}$ and from $C'$ to $C_{\text{accept}}$. Each level of recursion uses $O(f)$ space; depth is $O(f)$; total $O(f^2)$.

The canonical PSPACE-complete problem is TQBF — given a fully quantified boolean formula $\forall x_1\exists x_2\cdots Q_n x_n\,\varphi$, is it true? The alternating quantifiers correspond to alternating polynomial-space computation. Two-player games with polynomial-length play (geography, Reversi, generalised hex) reduce to TQBF.

6. Time and space hierarchy theorems

Most separations between complexity classes are open ($\mathsf{P}$ vs $\mathsf{NP}$, $\mathsf{NL}$ vs $\mathsf{P}$, $\mathsf{NP}$ vs $\mathsf{PSPACE}$). The hierarchy theorems are the few separations we can prove — and they all come from diagonalisation.

Time hierarchy (Hartmanis–Stearns 1965). If $f, g$ are time-constructible and $f(n)\log f(n) = o(g(n))$, then $\mathrm{DTIME}(f)\subsetneq\mathrm{DTIME}(g)$.

Sketch. Build $D$ that on input $\langle M\rangle$ simulates $M$ on $\langle M\rangle$ for at most $g(n)$ steps; if $M$ accepts within budget, $D$ rejects, and vice versa. The simulation overhead is the $\log f$ factor (universal-TM bookkeeping). $D$ runs in time $g$, so $L(D)\in\mathrm{DTIME}(g)$. If $L(D)\in\mathrm{DTIME}(f)$, some $M_0$ deciding it has time $\le f(n)$, hence $D$ disagrees with $M_0$ on $\langle M_0\rangle$ — contradiction.

Space hierarchy. If $f, g$ space-constructible with $f=o(g)$, $\;\mathrm{DSPACE}(f)\subsetneq\mathrm{DSPACE}(g)$ (no $\log$ factor — universal simulation in space is constant overhead).

Concrete separations:

  • $\mathsf{P}=\bigcup_k\mathrm{DTIME}(n^k)\;\subsetneq\;\mathrm{DTIME}(2^n)\subseteq\mathsf{EXP}$ — because $n^k\log n^k = o(2^n)$ for every $k$.
  • $\mathsf{L}=\mathrm{DSPACE}(\log n)\;\subsetneq\;\mathsf{PSPACE}$ — by space hierarchy with $f=\log n$, $g=n$.
  • $\mathsf{NL}\subsetneq\mathsf{NPSPACE}=\mathsf{PSPACE}$ — combining space hierarchy and Savitch.

What the hierarchy theorems do not give us is any separation across the determinism/nondeterminism boundary, or any "natural" separation: a polynomial-time language can be hard for $\mathrm{DTIME}(n^{100})$ but easy for $\mathrm{DTIME}(n^{200})$ — but no such language is known to be useful.
