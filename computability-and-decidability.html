Computability theory

What can a machine compute, and what is permanently out of reach? From Turing's tape and Kleene's recursion schemes to the diagonal that kills the halting problem, the gap between recursive and recursively enumerable, Rice's theorem, and the computability lens on Gödel — six concepts that fence off the reachable from the unreachable.

An algorithm is something we recognise when we see it, but pinning down the notion turns out to be the deepest move in 20th-century logic. Turing's tape, Church's $\lambda$-calculus, and Kleene's $\mu$-recursion all converged on the same class of partial functions $\mathbb{N}^k\rightharpoonup\mathbb{N}$. The Church–Turing thesis is the empirical claim that any reasonable model of computation lands at the same place. Once you accept this, a single counting argument — there are countably many algorithms but uncountably many functions — already guarantees most functions are not computable. The sharper question is which specific problems are out of reach: halting, equivalence, satisfiability, Hilbert's tenth, and (via reduction) most of the natural problems about programs themselves.

Plan. Section 1 fixes a model — the Turing machine — and Section 2 shows it agrees with recursion-theoretic schemes. Section 3 runs Cantor's diagonal against the universal simulator to kill halting. Sections 4–5 unpack the consequences: r.e. but not recursive sets, the many-one degree structure, Rice's theorem. Section 6 ports halting into arithmetic to recover Gödel's incompleteness theorems.

1. Turing machines

A (deterministic, single-tape) Turing machine is a tuple $M = (Q, \Sigma, \Gamma, \delta, q_0, q_{\text{acc}}, q_{\text{rej}})$ where $Q$ is a finite state set, $\Sigma\subset\Gamma$ are the input and tape alphabets (the blank $\sqcup\in\Gamma\setminus\Sigma$), and

\[ \delta\colon Q\times \Gamma \to Q\times\Gamma\times\{L,R\} \]

tells the machine, given its current state and the symbol under the head, which symbol to write, which way to move the head, and which state to enter next. A configuration is the triple $(q, T, h)$ recording state, full tape contents, and head position. The machine starts at $q_0$ with the input written on the leftmost cells and blanks elsewhere; it accepts if it reaches $q_{\text{acc}}$, rejects at $q_{\text{rej}}$, and loops if it never halts.

Robustness of the model. Multitape, two-counter, two-stack, RAM, and nondeterministic TMs all simulate one another with at most polynomial overhead. The class of computable functions is the same. This empirical convergence is the content of the Church–Turing thesis: any reasonable formalism for "effective procedure" yields exactly this class.

2. Recursive and partial-recursive functions

Turing's tape is one model; Kleene's recursion schemes are another. The class of partial recursive functions $\mathbb{N}^k\rightharpoonup\mathbb{N}$ is the smallest collection containing the basic functions:

  • Zero $Z(n) = 0$.
  • Successor $S(n) = n+1$.
  • Projections $P_i^k(x_1,\ldots,x_k) = x_i$.

and closed under three operations:

  • Composition. If $g\colon\mathbb{N}^m\to\mathbb{N}$ and $h_1,\ldots,h_m\colon\mathbb{N}^k\to\mathbb{N}$ are recursive, so is $g(h_1(\vec x),\ldots,h_m(\vec x))$.
  • Primitive recursion. Define $f$ by $f(\vec x, 0) = g(\vec x)$ and $f(\vec x, n+1) = h(\vec x, n, f(\vec x, n))$.
  • Unbounded minimisation ($\mu$). $f(\vec x) = \mu y\,[g(\vec x, y) = 0]$ — the least $y\ge 0$ with $g(\vec x, y) = 0$, undefined if no such $y$ exists.

Without the $\mu$-operator you get the primitive-recursive functions — every member is total, and addition, multiplication, exponentiation, and bounded search all live here. With $\mu$ you reach partial functions and exact agreement with Turing-computability.

Why $\mu$ is the bridge. Primitive recursion can simulate any bounded search, but Turing machines can run forever. The $\mu$-operator is the recursion-theoretic "while-loop": ask $g(\vec x,0), g(\vec x,1),\ldots$ until a zero appears, and accept partiality if no zero ever does. Ackermann's function is total and TM-computable but not primitive-recursive — you need $\mu$ (or a TM, equivalently) to define it cleanly.

3. The halting problem

Once we have a notion of computation, the natural meta-question is: can a program decide whether another program halts? Encode every Turing machine $M$ as a string $\langle M\rangle$ and form the language

\[ H \;=\; \{\,\langle M, w\rangle : M \text{ halts on input } w\,\}. \]

Suppose, for contradiction, $h$ is a TM that decides $H$: it always halts and accepts iff $\langle M,w\rangle\in H$. Build a new machine $D$ that on input $\langle M\rangle$:

  1. Runs $h$ on $\langle M, \langle M\rangle\rangle$.
  2. If $h$ says "halts", $D$ enters an infinite loop. If $h$ says "doesn't halt", $D$ halts.

What does $D$ do on its own code $\langle D\rangle$? If $D$ halts on $\langle D\rangle$, then $h$ on $\langle D,\langle D\rangle\rangle$ said "halts", so $D$ is supposed to loop — contradiction. If $D$ loops, $h$ said "doesn't halt", so $D$ should halt — contradiction. Either way we lose, so no such $h$ exists.

Theorem (Turing 1936). The halting problem $H$ is undecidable: no Turing machine decides it.

4. Recursive vs recursively enumerable

The halting problem is undecidable, but it is not "infinitely far" from being decidable. It is semi-decidable: simulate $M$ on $w$, and if it halts, accept; if it doesn't, you wait forever — but you do correctly classify every member of $H$. This is the next layer up.

A set $A\subseteq\mathbb{N}$ is recursive (decidable) if its characteristic function $\chi_A$ is computable; recursively enumerable (r.e., semi-decidable, $\Sigma_1^0$) if there is a TM that halts exactly on inputs in $A$. Equivalently: $A$ is the range of a (partial) computable function, or $A = \{x : \exists y\,T(e, x, y)\}$ for some Turing-machine code $e$ and Kleene's $T$-predicate.

Post's theorem. $A$ is recursive iff both $A$ and $\mathbb{N}\setminus A$ are r.e. Proof. If both have semi-deciders, dovetail them on each input — exactly one halts, giving a decider. Conversely a decider yields semi-deciders for both.

So the gap "recursive ⊊ r.e." is exactly the gap between r.e. and co-r.e. The halting set $H$ is r.e. (just simulate); not recursive (Turing's theorem); and its complement $\overline H$ is not even r.e. (a separate diagonal).

5. Undecidability via reduction

Reductions are the engine for spreading undecidability from one problem to another. For sets $A, B\subseteq\mathbb{N}$, write

\[ A \le_m B \quad\text{iff}\quad \exists\,\text{computable }f\colon\mathbb{N}\to\mathbb{N}\text{ with } x\in A \iff f(x)\in B. \]

Equivalently: a black-box decider for $B$ would, via $f$, decide $A$. The point is the contrapositive: if $A$ is undecidable and $A\le_m B$, then $B$ is undecidable too. Hardness travels upward.

Rice's theorem. Let $\mathcal{P}$ be a non-trivial property of r.e. languages (some TM has it, some doesn't). Then $\{\langle M\rangle : L(M)\in\mathcal{P}\}$ is undecidable. Sketch. WLOG $\emptyset\notin\mathcal{P}$; pick $M_0$ with $L(M_0)\in\mathcal{P}$. Given $\langle M,w\rangle$, build $M'$ that on input $x$ first simulates $M$ on $w$ (ignoring output), then runs $M_0$ on $x$. Then $L(M') = L(M_0)$ if $M$ halts on $w$, else $\emptyset$. So $\langle M'\rangle\in\mathcal{P}$ iff $\langle M,w\rangle\in H$ — a halting reduction.

The map $H\le_m B$ is the swiss-army knife. Use it to kill TM-emptiness, TM-equivalence, totality, regularity, context-freeness — and via more delicate reductions, Hilbert's tenth problem (Diophantine solvability), the word problem for finitely presented groups, and Post's correspondence problem.

6. Gödel incompleteness — computability lens

Gödel's incompleteness theorems (1931) predate Turing by five years, but in hindsight the cleanest proof routes through halting. Fix a consistent, recursively axiomatised theory $T$ extending Robinson arithmetic $\mathsf{Q}$ — strong enough to express bounded computation, weak enough that we know what its proof-search looks like.

Inside $T$ we can encode Turing machines, configurations, and runs as natural numbers (Gödel numbering). The statement "$M$ halts on $w$" becomes a $\Sigma_1$ formula:

\[ \mathrm{Halt}(\ulcorner M\urcorner, \ulcorner w\urcorner) \;\equiv\; \exists t\, \mathrm{Halts\text{-}by\text{-}step}(\ulcorner M\urcorner,\ulcorner w\urcorner, t). \]
First incompleteness theorem. If $T$ is consistent and recursively axiomatised, $T$ is incomplete: there is a true $\Sigma_1$ sentence (about halting) that $T$ neither proves nor refutes.

Why: if $T$ proved every true halting fact and refuted every false one, we could decide halting by alternating proof-search on $\mathrm{Halt}$ and $\neg\mathrm{Halt}$ — contradicting Turing. The "true but unprovable" sentence Gödel produces is exactly such a missing halting fact, dressed up via the diagonal lemma so it self-references its own unprovability.

Rosser's refinement. Gödel's original proof needed $\omega$-consistency. Rosser modified the self-referential sentence to "for every proof of me there is a shorter refutation", so the conclusion goes through with plain consistency.

The second incompleteness theorem formalises the first inside $T$: from $\mathrm{Con}(T)$ one can derive (within $T$) the unprovability of the Gödel sentence $G$, hence $G$ itself — contradicting $T\not\vdash G$. So $T\not\vdash\mathrm{Con}(T)$.

7. Connections

Once you accept the Church–Turing thesis, "what can be computed" stops being a soft question — it has a sharp answer, and the sharpness is what makes the rest of the theory work.

Complexity theory

Computability says whether a problem can be solved at all. Complexity theory says how efficiently — the same diagonal argument that gives the halting problem also gives the time / space hierarchy theorems.

First-order logic and Gödel

First-order logic + Gödel-arithmetisation reduces "$T$ proves $\varphi$" to a recursively enumerable but not decidable predicate; the halting problem reappears as the consistency-strength comparison theorem.

Type theory and proofs as programs

Type theory via Curry–Howard makes computability and provability the same activity: a term inhabiting type $A$ both witnesses the truth of $A$ and computes a value of type $A$.

Algorithms and number theory

Hilbert's tenth problem (Diophantine equations are undecidable) connects computability to number theory; analogously, the word problem in groups is undecidable for finitely-presented groups but decidable for hyperbolic groups.

Open frontiers: Kolmogorov complexity and Chaitin's $\Omega$ (incompleteness via algorithmic randomness); higher computability (recursion in higher types, hyperarithmetic / analytical hierarchy); reverse mathematics; computational interpretations of constructive proofs.
